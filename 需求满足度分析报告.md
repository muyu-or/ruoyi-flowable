# 需求满足度分析报告

> 分析日期：2026-05-10  
> 分析范围：RuoYi-Flowable v3.8.9 全模块代码库

---

## 一、库存管理模块 — 通过扫码枪具备出入库管理

| 子项 | 状态 | 说明 |
|------|------|------|
| 扫码入库 | ✅ 已实现 | `POST /manage/inventory/scan-inbound`，前端有扫码模式切换（`inventory/index.vue`），输入物料编码+数量后回车即可入库 |
| 扫码出库 | ❌ 缺失 | 仅有手动选择行 + 填数量的出库方式（`POST /manage/inventory/stock-out`），无 `scan-outbound` 接口，前端无扫码出库模式 |

**缺失详情：**
- 后端：`IInventoryService` / `InventoryServiceImpl` 无 `scanOutbound()` 方法
- 前端：`inventory/index.vue` 出库弹窗需先选中表格行，再手动输数量，无扫码枪输入框
- 前端 API：`inventory.js` 无 `scanOutbound()` 封装

## 五、权限管理模块

### 5.1 管理团队权限层级配置

| 状态 | 说明 |
|------|------|
| ❌ 缺失 | 无多级管理团队层级表，角色扁平（全部 `data_scope='1'`），无层级权限配置 |

**缺失详情：**
- 仅有 `sys_dept`（部门）作为组织架构，无专门的管理团队层级实体
- 角色之间无层级继承关系
- 无法配置"一级管理看全部，二级管理看本区"等层级权限
- 6 个自定义业务角色（warehouse / preprocess / device_operator / quality_inspector / team_leader / zongguan）全部 `data_scope='1'`

### 5.2 作业班组权限配置

| 状态 | 说明 |
|------|------|
| ⚠️ 部分 | 班组结构（`production_team` + `production_team_user`）已有，但班组无独立权限集 |

**缺失详情：**
- 已实现：班组 CRUD、成员管理、职位设置（班组长/操作员/质检员/设备员/安全员/技术员/助理）
- 已实现：通过 Flowable candidate 机制注入团队任务可见性（`FlowTeamServiceImpl`）
- 缺失：班组级独立权限配置（班组可见哪些菜单/功能/数据）
- 缺失：班组权限配置 UI
- 当前班组权限完全依赖成员个人的 `SysRole` 分配

### 5.3 后台管理员权限配置

| 状态 | 说明 |
|------|------|
| ⚠️ 部分 | 通过标准 RuoYi RBAC 实现（admin `*:*:*` / zongguan 全局权限），无独立管理面板 |

**缺失详情：**
- 已实现：admin 用户（userId=1）自动获得 `*:*:*` 通配符
- 已实现：生产总管（zongguan, role_id=100）通过角色菜单分配获得大部分业务权限
- 已实现：`PermissionsService` 支持 `hasPermi` / `hasRole` / `hasAnyPermi` 等
- 缺失：独立的管理员权限配置面板（可视化管理各管理员权限范围）
- 缺失：管理员操作审计日志

### 5.4 系统功能权限分配

| 状态 | 说明 |
|------|------|
| ✅ 已实现 | 标准 RuoYi RBAC：`@PreAuthorize("@ss.hasPermi(...)")` + `v-hasPermi` 指令 + 菜单权限字符串 |

现有功能：
- 后端：注解驱动的接口级权限控制
- 前端：`v-hasPermi` 指令控制按钮/元素可见性
- 前端：路由守卫 + Vuex 动态权限菜单生成
- 权限字符串规范：`manage:inventory:list/add/edit/remove/export`
- 所有业务控制器均有对应权限注解

### 5.5 数据管理权限管理

| 状态 | 说明 |
|------|------|
| ⚠️ 部分 | 仅有部门级 DataScope（5 级：全部/自定义/本部门/本部门及以下/仅本人），无班组/仓库级数据隔离 |

**缺失详情：**
- 已实现：`@DataScope` 注解 + `DataScopeAspect` AOP 切面实现部门数据隔离
- 已实现：5 级数据范围（`data_scope` = 1~5）
- 已实现：`PermissionsService` + `SecurityUtils` 工具类
- 缺失：按班组过滤数据（如班组长只能看本班组的生产数据）
- 缺失：按仓库/区域过滤库存数据
- 缺失：跨模块数据权限统一管理
- 缺失：数据权限的可视化配置界面

## 六、总结

### 完全缺失（需从零开发，共 6 项）

| 序号 | 需求 | 说明 |
|------|------|------|
| 1 | 扫码出库 | 后端 `scan-outbound` 接口 + 前端扫码出库模式 |

| 6 | 管理团队权限层级配置 | 多级管理层级表 + 层级权限 |
